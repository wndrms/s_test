# Data Model Notes

## 1. 핵심 원칙

이 시스템의 기준 데이터는 화면 값이 아니라 원장과 체결입니다.

- 보유현황은 `cash_ledger`, `trade_fills`, `positions`, `quote_snapshots`에서 계산합니다.
- 실현손익은 매도 체결 기준으로 계산합니다.
- 미실현손익은 최신 quote 기준으로 계산합니다.
- AI 분석은 주문의 원인이지만 원장 데이터를 직접 수정하지 않습니다.
- 주문 실행은 반드시 `order_plans` -> `broker_orders` -> `trade_fills` 순서로 기록합니다.

## 2. Manager = Account 1:1

확정 정책:

- 매니저 1개는 한국투자증권 계좌 1개와 1:1 매칭합니다.
- 모의 매니저는 PaperBroker를 사용합니다.
- 실전 매니저는 KIS real environment를 사용합니다.
- 모의/실전 주문 경로는 코드 레벨에서 분리합니다.

## 3. Symbol Mapping

한국과 미국을 동시에 지원하므로 종목 식별자는 반드시 분리 저장합니다.

예시:

```text
symbols.code = 005930
symbol_identifiers:
  KIS_CODE = 005930
  DART_CORP_CODE = 00126380
  ISIN = KR7005930003

symbols.code = AAPL
symbol_identifiers:
  KIS_CODE = AAPL
  CIK = 0000320193
```

## 4. Evidence Card

뉴스/공시/재무/커뮤니티 원문을 AI에 직접 대량으로 넣지 않습니다. 수집 데이터를 정규화한 뒤 `evidence_cards`로 압축합니다.

Evidence Card는 다음 조건을 만족해야 합니다.

- 어떤 소스에서 왔는지 명확해야 함
- 어떤 종목과 연결되는지 명확해야 함
- 데이터 시점 `as_of`가 있어야 함
- 중요도와 신뢰도 점수가 있어야 함
- 원문 저장 대신 요약/링크 중심이어야 함

## 5. 데이터 신선도

자동매매 필수 데이터:

- 가격 데이터
- 계좌/잔고 데이터
- 주문 가능 금액
- 장 운영 상태
- 휴장일 여부

위 데이터가 오래되었거나 없으면 실전 주문을 금지합니다.

보조 데이터:

- 뉴스
- 공시
- 재무제표
- 커뮤니티

보조 데이터가 오래되면 주문을 막을 수도 있지만, 기본은 risk score 감점과 분석 경고 표시입니다. 단, 특정 종목에 중대한 공시 이벤트가 감지되었는데 원문 확인에 실패하면 blocking으로 처리할 수 있습니다.

## 6. 삭제 정책

화면의 위험 구역에서 매니저 삭제를 누르더라도 실전 시스템에서는 hard delete를 피합니다.

권장:

- `managers.status = deleted`
- 자동매매 즉시 비활성화
- 연결 토큰 폐기 또는 비활성화
- 감사 로그 기록
- 주문/체결/분석 기록은 보존

## 7. 감사 로그

아래 이벤트는 반드시 `audit_logs`에 남깁니다.

- API 키 등록/수정/삭제
- 매니저 생성/수정/삭제
- 자동매매 활성화/비활성화
- 리스크 정책 변경
- 스케줄 변경
- AI 시나리오 생성
- 주문 계획 생성
- 주문 승인/거절
- 실전 주문 전송
- 체결 동기화
- 알림 발송 실패
